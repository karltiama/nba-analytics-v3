## Context
Player props ingestion runs via EventBridge Scheduler. We want an ET-based schedule (12:00–23:59 ET every 15 minutes) without manual UTC conversion / DST drift.

## Goal
Add a timezone variable for the player props Scheduler and update cron documentation to be timezone-interpreted.

## Files Changed
- `infra/variables.tf`

## Implementation Notes (key logic explained)
- Added `player_props_schedule_timezone` with default `America/New_York`.
- Updated `player_props_schedule_crons` description so cron expressions are interpreted using `player_props_schedule_timezone` (not assumed UTC).

## Key Decisions (and alternatives considered)
- Decision: use Scheduler timezone support to avoid DST headaches.
- Alternative: keep UTC-only crons and update twice per year; rejected as error-prone.

## Assumptions
- Terraform AWS provider supports `schedule_expression_timezone` for `aws_scheduler_schedule`.

## Failure Modes
- If the provider version is too old, `terraform plan` will fail and require a provider upgrade.

## How to Test (commands + steps)
1. `terraform -chdir=infra plan`
2. Confirm the new variable is recognized and no validation errors occur.

## Next Steps
- Update `infra/terraform.tfvars` to set timezone + 15-minute cron list.
- Run `terraform plan` and verify next invocations align with ET.
