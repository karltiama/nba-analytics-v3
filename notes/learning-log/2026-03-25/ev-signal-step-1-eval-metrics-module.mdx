## Context
We need a repeatable way to measure EV signal quality before changing API ranking behavior.

## Goal
Add shared evaluation metrics utilities for calibration and ranking quality checks.

## Files Changed
- `lib/betting/ev-eval-metrics.ts`

## Implementation Notes (key logic explained)
- Added `brierScore` to measure probabilistic forecast error.
- Added `expectedCalibrationError` and `calibrationBins` for bucket-based reliability checks.
- Added `decileScorecard` to evaluate EV ranking monotonicity and realized ROI by top-to-bottom deciles.

## Key Decisions (and alternatives considered)
- Chose lightweight in-repo metric functions instead of external stats dependencies for portability.
- Used simple fixed-width probability bins (default 10) to keep metrics interpretable.

## Assumptions
- Inputs to these functions are prevalidated rows with binary outcomes and finite probabilities.

## Failure Modes
- Sparse samples can produce noisy calibration bins and unstable decile ROI.
- If probabilities are not clamped, metrics can become misleading.

## How to Test (commands + steps)
1. Import functions in a script or unit test.
2. Feed synthetic rows with known outcomes.
3. Verify Brier and ECE change in expected direction when probabilities improve.

## Next Steps
- Use this module in an offline backtest script against settled player props.
