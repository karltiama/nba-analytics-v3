## Snapshot rule

**Closing-line proxy:** for each `(game_id, player_id, sportsbook, prop_type, side)`, the row with the **latest** `raw.player_prop_snapshots_v2.fetched_at` **strictly before** `analytics.games.start_time`. Intra-day history is only as complete as raw ingestion.

## Holdout filtering

Use query params on the API or UI:

- `after=YYYY-MM-DD` — `game_date >= after` (inclusive), typical **test** window start.
- `before=YYYY-MM-DD` — `game_date < before` (exclusive), typical **train** ceiling or test end.

## Assumptions

- `analytics.games.game_id` text matches `raw.player_prop_snapshots_v2.game_id::text`.
- Final games only; outcomes from `analytics.player_game_logs`.

## Failure modes

- Views missing in DB → API returns 503 with install hint.
- Sparse raw history → few eval rows (expected).

## How to test

1. In Supabase SQL Editor, run **`db/schemas/research_install_all.sql`** once (or the four separate `research_*.sql` files in order).
2. `npx tsx scripts/research-prop-eval-sample.ts --after YYYY-MM-DD`
3. Open `/betting/research` with dev server.
