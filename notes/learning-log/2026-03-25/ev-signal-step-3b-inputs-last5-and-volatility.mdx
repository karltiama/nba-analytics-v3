## Context
Track B needs richer features than baseline (`last10`, `season`) to improve probability quality.

## Goal
Extend model input extraction to include short-term trend (`last5`) and observed recent volatility (`std10`).

## Files Changed
- `lib/betting/player-prop-inputs.ts`

## Implementation Notes (key logic explained)
- Added `stddev()` helper and extended input payload with:
  - `ext.last5` per stat
  - `ext.std10` per stat
- Updated `getStatsForPropType()` to return:
  - `last10Avg`, `seasonAvg`, `last5Avg`, `observedStdDev`
- Added safe fallbacks when no games are available.

## Key Decisions (and alternatives considered)
- Kept features simple and directly derivable from existing game logs.
- Used sample stddev for stability and consistent scaling.

## Assumptions
- Last-10 and last-5 slices are ordered by recency in upstream game queries.

## Failure Modes
- Very low game counts can produce weak/zero variance estimates.
- Injury/role shifts can still cause lagging averages to misrepresent expectation.

## How to Test (commands + steps)
1. Run API route that calls `getStatsForPropType()`.
2. Confirm returned object now includes `last5Avg` and `observedStdDev`.
3. Validate no runtime errors for players with sparse history.
