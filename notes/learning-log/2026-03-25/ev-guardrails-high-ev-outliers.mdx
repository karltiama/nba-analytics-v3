## Context
Several props were showing EV values above 30%, likely driven by overconfident probabilities when projection sat far from the offered line, especially under dynamic sigma settings.

## Goal
Reduce unrealistic EV outliers by adding conservative probability guardrails while preserving model ranking signal.

## Files Changed
- `lib/betting/player-prop-model.ts`

## Implementation Notes (key logic explained)
- Added z-score cap in `computeProbability`:
  - `MAX_ABS_Z = 3.0`
  - Prevents extreme tails from producing near-0 or near-1 probabilities purely from projection-line gaps.
- Added probability clamp:
  - `PROB_FLOOR = 0.03`
  - `PROB_CEIL = 0.97`
- Added Track B sigma floor:
  - `stdDev = max(0.7 * fallbackStd, blendedStd)`
  - Prevents dynamic sigma collapse from making probabilities overly confident.

## Key Decisions (and alternatives considered)
- Decision: add deterministic guardrails in model math layer.
- Alternative considered: hard-cap EV at API layer.
  - Rejected because it hides model confidence issues instead of correcting them upstream.

## Assumptions
- Most problematic outliers come from tails of Normal CDF with small sigma and large line-projection deltas.
- A 3%/97% probability envelope is acceptable for current ranking use.

## Failure Modes
- Legitimate high-confidence edges may be slightly compressed.
- If calibration artifacts assume unbounded tails, recalibration may be needed after guardrail deployment.

## How to Test (commands + steps)
1. `npm run build`
2. Query props API with EV enabled and inspect top rows:
   - `/api/betting/props-explorer?date=<today>&sort=ev&dir=desc&limit=50`
3. Confirm fewer extreme `ev > 0.30` rows versus previous behavior.
4. Spot-check that ranking remains sensible for top positive EV rows.
