## Context
Track A/Track B calibration needed real fitted coefficients instead of identity defaults to measure practical signal improvements.

## Goal
Create a fitting script that learns per-prop linear calibration coefficients and writes artifacts consumed by runtime calibration.

## Files Changed
- `scripts/fit-ev-calibration.ts`

## Implementation Notes (key logic explained)
- Queries settled historical rows and reconstructs outcomes.
- Computes baseline and upgraded probabilities per row.
- Fits per-prop linear calibration (`y = slope * p + intercept`) with shrinkage toward identity.
- Writes artifact JSON to `lib/betting/ev-calibration-artifacts.json`.

## Key Decisions (and alternatives considered)
- Used linear calibration as a stable first pass before isotonic/platt complexity.
- Added shrinkage to reduce overfitting on limited history windows.

## Assumptions
- Historical settled rows are representative enough for coefficient fitting.

## Failure Modes
- Small sample prop types may produce noisy coefficients.
- Leakage risk exists if time windows are not handled carefully in future variants.

## How to Test (commands + steps)
1. Run: `npx tsx scripts/fit-ev-calibration.ts --days 45`
2. Confirm file `lib/betting/ev-calibration-artifacts.json` updates.
3. Re-run evaluator and compare Track A/B metrics vs baseline.
