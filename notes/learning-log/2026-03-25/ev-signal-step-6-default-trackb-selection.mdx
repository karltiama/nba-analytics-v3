## Context
We completed EV signal expansion and validated that calibrated Track B outperforms baseline and Track A on Brier and ECE. The app still defaulted to baseline when `EV_RANKING_TRACK` was unset.

## Goal
Set the default ranking model to calibrated Track B for API/UI behavior, while keeping Track A available as a diagnostic reference.

## Files Changed
- `lib/betting/ev-selection-policy.ts`

## Implementation Notes (key logic explained)
- Updated the `DEFAULT_TRACK` constant from `baseline` to `trackB_calibrated`.
- Kept all explicit environment overrides unchanged:
  - `trackA_calibrated` aliases still resolve to Track A.
  - `trackB_calibrated` aliases still resolve to Track B.
  - `baseline` still resolves when explicitly requested.
- Result: if `EV_RANKING_TRACK` is missing or invalid, the API now selects Track B for `ev` / `modelProbability`.

## Key Decisions (and alternatives considered)
- Decision: change only the default selector constant.
- Alternative considered: force `EV_RANKING_TRACK` in environment for every deployment.
  - Rejected because it is operationally brittle and still leaves undefined fallback behavior.

## Assumptions
- UI continues using API `ev` as the primary ranking signal.
- Track A should remain visible via diagnostic fields (`evTrackA`, `modelProbabilityTrackA`) and not be removed.

## Failure Modes
- If a deployment explicitly sets `EV_RANKING_TRACK=baseline`, Track B will not be used.
- If consumers incorrectly rely on fallback baseline behavior, ranking/order may change unexpectedly after this update.

## How to Test (commands + steps)
1. Build:
   - `npm run build`
2. API default behavior (without env override):
   - Request: `/api/betting/players/175/props?game_id=18447872&with_consensus=1&with_ev=1`
   - Verify response has `evSelectedTrack: "trackB_calibrated"`.
3. Diagnostic preservation:
   - Verify `evTrackA` and `modelProbabilityTrackA` are still present in each prop row.
4. Optional explicit override checks:
   - Set `EV_RANKING_TRACK=trackA_calibrated`, restart dev server, confirm selected track changes.
   - Set `EV_RANKING_TRACK=baseline`, restart dev server, confirm selected track changes.

## Next Steps
- Keep Track B as production default.
- Run weekly calibration/evaluation checks to ensure Track B remains best.
- If Track B degrades, temporarily switch to Track A via env var while recalibrating.
