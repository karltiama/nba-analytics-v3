## Context
We need to validate EV/calibration quality offline before exposing any new ranking logic in the API.

## Goal
Create an offline evaluation harness that computes baseline and upgraded track scorecards from settled games.

## Files Changed
- `scripts/evaluate-ev-signal.ts`

## Implementation Notes (key logic explained)
- Added a SQL pipeline to pull settled over/under rows and join realized player stats.
- Computes model probabilities for baseline and upgraded tracks.
- Calculates Brier, ECE(10), and EV decile scorecards for both tracks.
- Outputs a structured JSON summary for quick compare snapshots.

## Key Decisions (and alternatives considered)
- Chose script-first evaluation so model iteration is fast and decoupled from API runtime.
- Kept inputs tied to existing analytics tables to avoid new ETL dependencies.

## Assumptions
- `SUPABASE_DB_URL` is available in local `.env`.
- Settled games and player logs are populated in analytics schema.

## Failure Modes
- Missing historical rows can reduce sample size and weaken significance.
- Heavy query windows may run slowly; default window is limited by `--days`.

## How to Test (commands + steps)
1. Ensure `.env` contains `SUPABASE_DB_URL`.
2. Run: `npx tsx scripts/evaluate-ev-signal.ts --days 21`
3. Inspect JSON for:
   - `baseline.brier`, `upgraded.brier`
   - `baseline.ece10`, `upgraded.ece10`
   - decile trend shape.
