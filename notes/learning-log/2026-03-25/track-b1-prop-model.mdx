## Implementation Notes (key logic explained)

- **Stability**: From last 10 `GameLog` rows, per stat series: games with valid minutes + stat, minutes CV, stability score `1 - min(1, cv)`, and stat coefficient of variation on the series.
- **L5 weight**: Capped at `W_L5_MAX` (0.45), shrunk by low sample count, high minutes CV, high stat CV, and extra factor for combos.
- **σ**: Blends observed with fallback, reliability-interpolates toward fallback, floor `0.7 * fallback`, then applies `COMBO_SIGMA_MULT` for combo props.
- **EV ladder**: Track B uses `computeTrackB1PlayerPropProbability` with per-prop stability; raw side prob → `calibrateProbability(..., trackB)` → `anchorToMarket`. Track A uses the same ladder on **unanchored** baseline side probability (order: raw → cal → anchor). Production `modelProbabilityTrackB` / `evTrackB` remain the **anchored** Track B values.
- **Offline scripts**: SQL `CASE` branches extended for `points_assists` / `points_rebounds` / `rebounds_assists` (and `pa`/`pr`/`ra`). `computeUpgradedPlayerPropProbability` without game-level stability uses **neutral** defaults inside the model wrapper.

## Key Decisions (and alternatives considered)

- **Track A calibration input**: Switched to calibrating **raw** baseline side probability then anchoring (plan-aligned ladder). This can shift Track A vs the previous “calibrate anchored baseline” path.
- **Player props API without `with_ev`**: New ladder/metadata keys are omitted (only legacy nine EV fields stay null) to avoid bloating every row.
- **`fit-ev-calibration` output path**: Uses `join(process.cwd(), 'lib/betting/ev-calibration-artifacts.json')` instead of a hard-coded absolute path.

## Assumptions

- `GameLog.minutes` is numeric after analytics mapping; policy parser also accepts string minutes defensively.
- DB `prop_type` values align with `PROP_TO_STAT` / SQL aliases (`pa`, `pr`, `ra`).

## Failure Modes

- **Calibration drift**: Raw Track B probabilities changed; refit `ev-calibration-artifacts.json` when ready.
- **Sparse combos**: Low `sampleGamesUsed` or missing stat series → heavy shrink toward fallback σ and low L5 weight.
- **Non–over/under rows**: `computePropEvFields` returns empty-shaped fields (nulls + defaults).
