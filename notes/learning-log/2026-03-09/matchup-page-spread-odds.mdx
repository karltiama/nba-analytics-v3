# Matchup Page: Add Spread Odds to Quick Betting Card

## Context

The matchup game details page has a "Quick Betting" card showing Spread, Moneyline, and Total. The Spread cell showed only the line (e.g. "CLE -13.5") without the odds. The Total cell already shows O/U odds on a second line; the user wanted the same for spread (home/away spread odds).

## Goal

Show spread odds (home and away) in the Spread column of the Quick Betting card on the matchup page, mirroring the Total section’s "O +100 / U -120" pattern.

## Files Changed

- `app/api/betting/games/[gameId]/details/route.ts` — add `spreadOddsHome` and `spreadOddsAway` to `currentOdds` response.
- `components/betting/MatchupPageLayout.tsx` — extend `CurrentOdds` type and render spread odds below the spread line when present.

## Implementation Notes

- **API**: `getGameOdds()` already returns `odds.home.spreadOdds` and `odds.away.spreadOdds`. The details route now passes these as `spreadOddsHome` and `spreadOddsAway` in `currentOdds`.
- **UI**: In the Spread cell we keep the main line (e.g. "CLE -13.5"). When either spread odds is non-null, we render a second line: `{HOME_ABBR} ±XXX / {AWAY_ABBR} ±XXX` using the same formatting as moneyline/total (e.g. -110, +105). Styling matches the Total odds line: `text-[10px] text-muted-foreground font-mono mt-0.5`.

## Key Decisions

- **Naming**: Used `spreadOddsHome` / `spreadOddsAway` in the API and `CurrentOdds` to align with existing `moneylineHome` / `moneylineAway` and to avoid confusion with the spread line value.
- **Optional display**: Odds line is only rendered when at least one of `spreadOddsHome` or `spreadOddsAway` is non-null, so older responses or missing data don’t show an empty odds row.

## Assumptions

- `game_odds_current` (or equivalent) supplies `home_spread_odds` and `away_spread_odds`; no new env or dependencies.

## Failure Modes

- If the DB has no spread odds for a game, the second line simply doesn’t appear (graceful degradation).
- If the API is cached or an old client doesn’t expect the new fields, extra fields are ignored; no breaking change.

## How to Test

1. Start dev: `npm run dev`.
2. Open `/betting`, click a game to open its matchup details page.
3. Confirm the Spread cell shows the line (e.g. "CLE -13.5").
4. If the game has spread odds in `analytics.game_odds_current`, confirm a second line appears: e.g. "CLE -110 / BOS -110".
5. For a game with no spread odds in DB, confirm only the spread line appears and no second line.
