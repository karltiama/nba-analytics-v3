## Context

The betting dashboard GameCard component was displaying literal `0` values for spread, moneyline, and a fake `O/U 220` default for games without odds data. This happened because:
1. Completed ("Final") games have no odds stored in `analytics.game_odds_current`
2. The API route coerced `null` → `0` via `?? 0`
3. The frontend `transformGame` added fake defaults like `|| 220` and `|| -110`
4. The GameCard rendered those zeros as if they were real odds

## Goal

Display "—" and "No odds available" for games without odds data, while correctly rendering real odds for upcoming games.

## Files Changed

- `components/betting/GameCard.tsx` — Updated `formatOdds()` and `formatSpread()` to return "—" for zero values; added `hasOdds` flag; conditionally render implied probability bar vs "No odds available"
- `app/betting/page.tsx` — Removed fake defaults (`|| 220`, `|| -110`) from `transformGame`; all fallbacks now go to `0` so GameCard's zero-check works consistently

## Implementation Notes (key logic explained)

- **Why check for 0?** In sports betting, moneyline is never 0 (valid values are negative favorites or positive underdogs). Spread 0 is theoretically possible ("pick 'em") but extremely rare and not in our data. Total 0 is impossible. So `=== 0` safely means "no data."
- **`hasOdds` flag**: Derived from `game.homeOdds.moneyline !== 0 || game.awayOdds.moneyline !== 0`. Controls whether to show the implied probability bar or a "No odds available" message.
- **Implied probability bar**: Only shown when real odds exist. Prevents misleading 50/50 displays for completed games.

## Key Decisions (and alternatives considered)

| Decision | Alternative | Why |
|----------|------------|-----|
| Check for `=== 0` in display layer | Pass `null` through the entire pipeline | Minimal changes; 0 is never a real odds value |
| Remove `220` and `-110` defaults from transformGame | Keep defaults, add separate "hasOdds" flag from API | Fake defaults masked missing data—better to surface it |
| Show "No odds available" text | Hide entire odds section | Users should know odds exist for some games, just not this one |

## Assumptions

- Moneyline of exactly 0 never occurs in real NBA betting odds
- The BDL odds pipeline is already populating `analytics.game_odds_current` for upcoming games
- Completed games that were never captured during pre-game will remain without odds

## Failure Modes

- If a real pick'em spread (0) occurs, it will display as "—" instead of "PK". This is an acceptable trade-off given our data source.
- If the odds pipeline stops running, all games will show "No odds available"

## How to Test (commands + steps)

1. Ensure dev server is running: `npm run dev`
2. Navigate to `/betting`
3. Verify completed games (MIL/ORL, SAS/HOU, etc.) show "—" for spread, ML, and total instead of `0`
4. Verify completed games show "No odds available" instead of a 50/50 probability bar
5. Verify upcoming games (CLE/PHI, BKN/MEM, OKC/DEN) show real odds values (e.g., CLE -650 / PHI +470)
6. Verify upcoming games show the correct implied probability bar
