## Context
The PlayerTrendChart (540px fixed width) sat in a ~1221px container, leaving ~680px of
dead space. The BettingLinePanel (line analysis) was a separate card below the chart,
and the hover tooltip appeared below the chart SVG.

## Goal
On larger screens (xl+), use the empty space to the right of the chart to display:
1. Hover game details (value, opponent, game number)
2. The line analysis panel (input + hit rate / margin / streak stats)

On smaller screens, keep everything stacked vertically.

## Files Changed
- `app/betting/players/[playerId]/components/PlayerTrendChart.tsx`
- `app/betting/players/[playerId]/components/BettingLinePanel.tsx`
- `app/betting/players/[playerId]/components/PlayerAnalysisClient.tsx`

## Implementation Notes

### PlayerTrendChart
- Added `children?: ReactNode` prop to accept embedded content for the right panel.
- Body layout changed to `flex flex-col xl:flex-row xl:items-start gap-4`.
- Left side: `shrink-0` wrapper holds the SVG + a small-screen-only tooltip (`xl:hidden`).
- Right side: `flex-1 min-w-0` column holds an xl-only hover card (`hidden xl:block`)
  and `{children}` (the line analysis).
- The hover tooltip renders in two places but only one is visible at a time via
  responsive display classes.

### BettingLinePanel
- Added `embedded?: boolean` prop.
- Extracted the inner content (input + stats grid) into a shared `content` variable.
- When `embedded=true`: renders with a subtle left border + compact header, no glass-card
  wrapper. Stats grid uses `grid-cols-2` (fits the narrower right panel).
- When `embedded=false`: renders the original full card (backward-compatible).

### PlayerAnalysisClient
- Removed the separate `<section>` for BettingLinePanel.
- Wrapped `<BettingLinePanel embedded />` as a child of `<PlayerTrendChart>`.

## Key Decisions (and alternatives considered)
| Decision | Why | Alternative |
|---|---|---|
| `children` prop instead of explicit `sidePanel` | Simpler API, standard React pattern | Named prop would be more explicit but adds coupling |
| Duplicate hover tooltip with responsive visibility | Avoids complex portal/absolute positioning | Could use a single tooltip that repositions via CSS, but more fragile |
| `embedded` prop on BettingLinePanel | Keeps backward compatibility, clear intent | Could split into two components, but increases file count |
| `grid-cols-2` for embedded stats | Right panel is ~580px, 4 cols would be too cramped | Could use a vertical list, but 2x2 grid is denser |

## Assumptions
- The xl breakpoint (1280px) is wide enough for both the 540px chart and a useful
  right panel (~580px after padding/gaps).
- The BettingLinePanel's `embedded` mode is only used inside PlayerTrendChart; no
  other consumers need it yet.

## Failure Modes
- On screens between 1024-1279px (lg but not xl), layout is stacked — the right panel
  content appears below. This is fine but creates a long vertical scroll.
- If chart width increases beyond 540px, the right panel gets squeezed. The `min-w-0`
  prevents overflow but stats may wrap.

## How to Test (commands + steps)
1. `npm run dev` → navigate to a player's betting page.
2. **xl+ screen (≥1280px):** Chart on left, hover card + line analysis on right.
3. **Hover a data point:** Right panel shows large value, metric, opponent, game number.
4. **Enter a line (e.g. 24.5):** Stats grid appears in the right panel as a 2×2 grid.
5. **Resize below xl:** Layout stacks — tooltip appears below chart, line analysis below that.
6. **Clear the line:** Placeholder text reappears in the stats area.
