# Player Page Theme Alignment with Betting Dashboard

## Context
The `/betting/players/[playerId]` page had a visually inconsistent look compared to the main `/betting/page.tsx`. Cards were plain `glass-card rounded-xl` without the structured header zones, colored border accents, pill badges, and navigation bar used on the betting dashboard.

## Goal
Align the player analysis page styling with the betting dashboard's card and section patterns so both pages feel like part of the same product.

## Files Changed
- `app/betting/players/[playerId]/page.tsx` — added sticky mini header bar with logo + badge
- `app/betting/players/[playerId]/components/PlayerHeader.tsx` — restructured with `border-l-4`, header zone, avatar, pill badge
- `app/betting/players/[playerId]/components/PlayerAnalysisClient.tsx` — added section headers with subtitles and "DATA-DRIVEN" badges
- `app/betting/players/[playerId]/components/PlayerTrendChart.tsx` — added card header zone with legend
- `app/betting/players/[playerId]/components/BettingLinePanel.tsx` — added `border-l-4 border-l-[#ff6b35]` accent and card header zone
- `app/betting/players/[playerId]/components/GameLogTable.tsx` — added `border-l-4 border-l-[#39ff14]` accent and card header zone with count badge

## Implementation Notes (key logic explained)
- **Card pattern**: Every card now follows `glass-card rounded-xl border-l-4 border-l-[color] overflow-hidden` + an inner `px-5 py-2.5 border-b border-white/5 bg-white/[0.02]` header zone + a `p-5` body zone. This mirrors `GameCard.tsx`.
- **Border colors**: Cyan (`#00d4ff`) for PlayerHeader, orange (`#ff6b35`) for BettingLinePanel, green (`#39ff14`) for GameLogTable — providing visual hierarchy.
- **Badge pills**: `text-[10px] px-2 py-0.5 bg-[color]/20 text-[color] rounded-full font-semibold` — the same pattern used for "CLOSE MATCHUP", "DATA-DRIVEN" etc on the betting page.
- **Mini header**: A lighter version of the betting `Header` component — sticky, glass-card, logo with pulse dot, "PLAYER PROFILE" badge. Uses server-compatible markup (no client state needed).

## Key Decisions (and alternatives considered)
| Decision | Alternative | Why |
|----------|-------------|-----|
| Lightweight mini header instead of reusing full `Header` component | Refactor `Header` to work in server components | Full header requires client state (date picker, theme toggle) — not relevant on the player page |
| Each card gets its own border-l color | Single accent color for all | Different colors per section creates visual hierarchy and matches the variety on the betting page |
| Kept `bg-white/[0.02]` syntax | Use `bg-white/2` (newer Tailwind suggestion) | Consistency with existing betting components that all use `/[0.02]` |

## Assumptions
- The player page doesn't need the date picker or theme toggle (those are betting dashboard-specific features)
- The `neon-text-cyan` and `pulse-dot` CSS classes exist in globals.css (they're used on the betting page)
- The `Zap` icon from lucide-react is already available (used in the main Header)

## Failure Modes
- If `neon-text-cyan` or `pulse-dot` CSS classes don't exist, the mini header logo text and green dot won't animate (graceful degradation)
- On very narrow screens, the PlayerHeader stat boxes may wrap — the flex layout handles this
- The `overflow-hidden` on cards with `border-l-4` is required; without it the border radius clips oddly on the left edge
