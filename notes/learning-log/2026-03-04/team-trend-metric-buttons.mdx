# Team trend chart: metric buttons (Team total, Game total, Spread, Money line)

## Context
Team page trend chart had only timeframe and location controls. User wanted a **stat-type selector as buttons** (like the player page’s Points / Rebounds / Assists), with four options: Team total, Game total, Spread (later), Money line (later).

## Goal
- Add a button group in the trend chart header: **Team total**, **Game total**, **Spread**, **Money line**.
- Implement **Team total** (current: team PPG) and **Game total** (team + opponent points per game).
- Leave **Spread** and **Money line** as disabled/placeholder for later.

## Files Changed
- `app/teams/[teamId]/components/TeamTrendChart.tsx` – New `TeamTrendMetric` type, `METRIC_BUTTONS`, `metric` / `onMetricChange` props, button row in header, tooltip label via `getMetricLabel`.
- `app/teams/[teamId]/components/TeamPageContent.tsx` – `trendMetric` state, import `TeamTrendMetric`, derive `trendData` and `seasonAvg` by metric (team_total, game_total, spread, moneyline placeholder), pass `metric` and `onMetricChange` to chart.

## Implementation Notes
- **Team total:** `data = points_for`, `seasonAvg = season_stats.points_for` (or avg of filtered).
- **Game total:** `data = points_for + points_against` per game, `seasonAvg = team PPG + opp PPG` when available.
- **Spread:** Data prepared as `margin` (or `points_for - points_against`) and avg; button is `disabled: true` until “implement later”.
- **Money line:** Button disabled; data fallback reuses team_total so chart still receives valid props.
- Button styling matches player `StatTabs`: active = cyan fill, inactive = glass; disabled = opacity + cursor-not-allowed and “Coming soon” title.

## Key Decisions
- **Spread/Money line disabled:** Per user, both are “implement later”; buttons are visible but non-clickable.
- **Legend:** “Avg X.X PPG” changed to “Avg X.X” so it works for pts, total pts, and margin without “PPG” for game total/spread.
- **Spread data:** Logic is in place (margin + avg) so enabling the button later only requires removing `disabled: true`.

## Assumptions
- `recent_form` games include `points_against` and `margin` (or we derive margin); API already provides these.
- Season stats may expose `points_against` for game-total season avg; otherwise we use the average of the filtered game totals.

## Failure Modes
- If `points_against` is missing on some games, game total uses 0 for that game (same as existing optional chaining).
- If `season_stats.points_against` is missing, game-total season avg falls back to the average of the displayed game totals.

## How to Test
1. Open a team page (e.g. `/teams/GSW`).
2. In the trend chart header, confirm four buttons: **Team total**, **Game total**, **Spread**, **Money line**.
3. Click **Team total** → chart shows team PPG; tooltip shows “X pts”.
4. Click **Game total** → chart shows (team + opp) points per game; legend “Avg X.X” is the game-total average.
5. **Spread** and **Money line** are disabled (greyed, not clickable); hover shows “Coming soon”.
6. Timeframe (L5/L10/L20/Season) and Location (All/Home/Away) still work for both Team total and Game total.
