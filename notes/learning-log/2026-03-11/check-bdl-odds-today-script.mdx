# Check BDL Odds Today Script

## Context

The app already has an odds pipeline: `lambda/odds-pre-game-snapshot` calls BallDontLie `/v2/odds` for a date (default today ET), stores raw snapshots, and transforms to analytics. There was no lightweight way to only *check* if odds are available for today without running the full Lambda (which requires DB and writes).

## Goal

Add a small script that calls BallDontLie `/v2/odds` for today (or a given date), parses the response, and prints whether odds are available plus a short summary. No database writes.

## Files Changed

| File | Action | Purpose |
|------|--------|---------|
| `scripts/check-bdl-odds-today.ts` | Created | API-only check for BDL odds on a date |

## Implementation Notes (key logic explained)

- **Date:** Today in ET via `toLocaleDateString('en-CA', { timeZone: 'America/New_York' })`. Optional CLI arg: `YYYY-MM-DD`.
- **Single request:** One GET to `https://api.balldontlie.io/v2/odds?dates[]=DATE&per_page=100`. No cursor follow (enough for a quick “any odds?” check).
- **Zod:** Same response shape as the Lambda (`data` array, optional `meta.next_cursor`) so we reuse a minimal schema.
- **Output:** Date, “Odds available: Yes/No”, total rows, unique game IDs, vendors, and up to 10 game IDs.

## Key Decisions (and alternatives considered)

| Decision | Why | Alternative |
|----------|-----|-------------|
| No DB, no cursor | Script is for a quick check only | Full Lambda run or cursor pagination for full count |
| Same env var as Lambda | `BALLDONTLIE_API_KEY` / `BALDONTLIE_API_KEY` | New env var — unnecessary |
| Optional date arg | Lets you check a specific date | Today only — less flexible |

## Assumptions

- `BALLDONTLIE_API_KEY` (or `BALDONTLIE_API_KEY`) is set in `.env` and has GOAT tier for `/v2/odds`.
- First page (100 rows) is enough to answer “are odds available for this date?”

## Failure Modes

| Scenario | Handling |
|----------|----------|
| Missing API key | Exit 1 with message to set in .env |
| 429 rate limit | Exit 1 with “try again later” |
| Non-200 response | Log status and body, exit 1 |
| Unexpected JSON shape | Zod safeParse, log error and slice of JSON, exit 1 |

## How to Test (commands + steps)

```bash
# Today (ET)
npx tsx scripts/check-bdl-odds-today.ts

# Specific date
npx tsx scripts/check-bdl-odds-today.ts 2026-03-12
```

Expect: “Odds available: Yes/No”, total rows, unique games, vendors. No DB or Lambda run.
