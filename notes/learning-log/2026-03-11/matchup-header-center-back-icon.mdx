# Matchup header: center content + back icon in same container

## Context
The matchup page had a standalone "Back to Betting" row above the sticky header card, and the header content (time + team matchup + section tabs) was left-aligned.

## Goal
- Center the sticky header content (time + matchup row and section tabs).
- Move the back control into the same sticky card as an icon-only button to free space.

## Files Changed
- `components/betting/MatchupPageLayout.tsx`

## Implementation Notes
- Removed the separate `<div className="mb-6">` that wrapped the "Back to Betting" button.
- Inside the sticky card’s first row: added a back button with `absolute left-4 top-1/2 -translate-y-1/2` so it sits on the left without affecting layout; used `aria-label="Back to Betting"` for accessibility.
- First row: `flex items-center justify-center` so the time + matchup block is centered; the absolute back button doesn’t shift the center.
- Tabs row: added `justify-center` so section tabs are centered.

## Key Decisions
- **Back as icon only:** Saves space and keeps a single compact header; back action is still clear via icon + aria-label.
- **Absolute positioning for back:** Keeps the main content truly centered without an extra spacer div.

## Assumptions
- `ArrowLeft` from lucide-react is already imported in this file.
- Navigate back via `router.push('/betting')` is correct.

## Failure Modes
- Small viewports: back icon and centered content could feel tight; consider hiding time or abbreviating on very small screens if needed later.

## How to Test
1. Open a matchup page (e.g. from `/betting` click a game).
2. Confirm the sticky header shows: back arrow on the left, time + team matchup centered, section tabs centered below.
3. Click back arrow → should go to `/betting`.
4. Scroll down → header stays sticky with same layout.

## Next Steps
