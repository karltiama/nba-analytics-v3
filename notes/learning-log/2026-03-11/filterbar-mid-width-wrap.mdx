# FilterBar: better layout for 640–950px (search + buttons)

## Context
Between 640px and ~950px the FilterBar looked bad: the search and the buttons to its right (sort, Favorites, Close Matchups, filters icon) were cramped in one row.

## Goal
Improve layout in the 640–950px range so the search bar and right-side buttons style well (search stays usable; controls don’t squeeze).

## Files Changed
- `components/betting/FilterBar.tsx`

## Implementation Notes
- **Wrapper for search + sort + filters:** Search, sort dropdown, and filter toggles are now inside one flex container with `flex-1 min-w-0 flex-wrap items-center gap-2 sm:gap-3`. So from 640px up they can wrap to a second row when there isn’t enough space.
- **Search width:** Search wrapper uses `min-w-[200px] w-full lg:min-w-0 lg:w-auto`. Below `lg` (1024px): search takes full width of its row so it stays usable, and sort + filter buttons wrap to the next line. At `lg` and up: search shares the row with sort/filters again (`min-w-0`, `w-auto`, flex-1).
- **Sort dropdown:** Kept in its own `relative` div so the dropdown still positions under the sort button. Added `whitespace-nowrap` on the sort and filter buttons so labels don’t break.
- **Filter toggles:** Restored as a sibling group next to the sort button (both inside the same wrapping flex), with `shrink-0 flex-wrap` so they can wrap only when the whole group wraps.

## Key Decisions
- **Wrap at 640–950:** Let the “search + sort + filters” group wrap so the second row is “sort + filters” and the first row is “search” full width, instead of one squeezed row.
- **`lg` (1024px) for single row:** Single-row layout (search + sort + filters) only from 1024px up; between 640 and 1024 we allow two rows for a cleaner look.

## Assumptions
- Tailwind `lg` is 1024px; no custom breakpoint added.

## Failure Modes
- Very narrow (e.g. 640px) with date nav: date + divider + inner; inner wraps so row 1 = search (full width of inner), row 2 = sort + filters. Layout holds.

## How to Test
1. Open `/betting` and set viewport to 640px, 768px, 900px: confirm search is full width on its row and sort + filter buttons sit on the row below.
2. Set viewport to 1024px+: confirm one row (search, sort, filters) with search taking remaining space.
3. With and without date nav, confirm no overflow and readable spacing.

## Next Steps
- None. Optional: tune `min-w-[200px]` or use a custom breakpoint (e.g. 950px) if you want single-row at 950px.
