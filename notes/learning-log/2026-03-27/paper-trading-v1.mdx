## Context

Paper Trading v1 adds a personal bet log from Props Explorer, settlement against Final box scores, and basic ROI.

## Goal

Ship DB-backed paper bets, API (CRUD list + settle), Props Explorer “Add”, Paper Bets page (open / history / summary), and nav.

## Files Changed

- `db/schemas/paper_schema.sql` — `paper.bets` table + indexes
- `lib/betting/paper-settlement.ts` — American profit, prop_type → stat, over/under evaluation
- `lib/betting/__tests__/paper-settlement.test.ts` — vitest coverage
- `app/api/betting/paper-bets/route.ts` — GET list, POST create
- `app/api/betting/paper-bets/settle/route.ts` — POST settle open bets
- `app/betting/paper/page.tsx` — UI + Suspense
- `app/betting/props-explorer/page.tsx` — Add column
- `components/betting/Header.tsx` — Paper link
- `app/betting/layout.tsx` — header on `/betting/paper`

## Implementation Notes (key logic explained)

- Bets store `game_id` / `player_id` as **text** to align with `analytics.games` and `research.v_player_game_outcomes`.
- Settlement joins `paper.bets` → `analytics.games` (Final) → `research.v_player_game_outcomes` for stats.
- Non–`over_under` markets settle as **void** with 0 profit.
- Unknown `prop_type` mapping → void.
- ROI summary: `sum(profit_units) / sum(stake_units)` on settled rows (client-side from GET settled).

## Key Decisions (and alternatives considered)

- No auth: single-tenant; document risk until RLS/auth.
- Duplicates allowed (same leg can be added twice).
- Milestone / non-O/U: void instead of blocking add.

## Assumptions

- `research.v_player_game_outcomes` exists in Supabase (same as research install).
- User runs `paper_schema.sql` before using the feature.

## Failure Modes

- Missing `paper.bets` → API 503 with message.
- Missing research view → settle 503.
- Final game but no player box row → bet stays open (`skippedNoBoxScore`).

## How to Test (commands + steps)

1. Apply SQL: run `db/schemas/paper_schema.sql` in Supabase SQL editor.
2. `npm run dev` → Props Explorer → **Add** on a row → lands on `/betting/paper?tab=open`.
3. After games Final, **Settle now** → history + P/L + summary ROI.
4. `npm test` — settlement unit tests.
