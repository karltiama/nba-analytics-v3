## Context

We needed **account-level spend** from AWS (totals and breakdown by service) using only the **AWS CLI**, not the console. Cost Explorer exposes this through the **`ce`** (Cost Explorer) service. This note records the exact commands, flags, and pitfalls used to produce the March 2026 sample reports.

## Goal

- Document how to query **BlendedCost** for a date range.
- Document how to **group costs by AWS service** via `get-cost-and-usage`.
- Capture **prerequisites**, **correct `--group-by` syntax** (AWS CLI v2 on Windows), and common errors.

## Implementation Notes (key logic explained)

### Prerequisites

1. **Cost Explorer enabled** in the account: Billing console → **Cost Explorer** → launch/enable. The CLI cannot enable it for you.
2. **IAM**: The caller needs Cost Explorer permissions (e.g. `AWSBillingReadOnlyAccess` or `ce:GetCostAndUsage`).
3. **Region**: Pass **`--region us-east-1`** for `aws ce` calls.
4. **Data lag**: After first enable, data can take **~24 hours** to appear; until then you may see `DataUnavailableException`.

### Time period

`--time-period` uses **`Start` inclusive** and **`End` exclusive**. Example for “March 1 through March 27”:

- `Start=2026-03-01,End=2026-03-28`

### Monthly total (BlendedCost)

```powershell
aws ce get-cost-and-usage `
  --time-period Start=2026-03-01,End=2026-03-28 `
  --granularity MONTHLY `
  --metrics BlendedCost `
  --region us-east-1 `
  --output json
```

Response shape: `ResultsByTime[].Total.BlendedCost.Amount` (USD string). The API may set **`Estimated": true`** for the current month until it finalizes.

### Breakdown by service

Use **`--group-by`** with type **`DIMENSION`** (all caps), not `Dimension`:

```powershell
aws ce get-cost-and-usage `
  --time-period Start=2026-03-01,End=2026-03-28 `
  --granularity MONTHLY `
  --metrics BlendedCost `
  --group-by Type=DIMENSION,Key=SERVICE `
  --region us-east-1 `
  --output json
```

Non-zero amounts appear under `ResultsByTime[].Groups[].Metrics.BlendedCost.Amount`; each group has `Keys` (e.g. `"AWS Lambda"`).

**Wrong:** `Type=Dimension,Key=SERVICE` → `ValidationException: Group Definition type is invalid` (observed with AWS CLI 2.22.x on Windows).

### Human-readable output

Add `--output table` to either command if you prefer console tables instead of JSON.

## Key Decisions (and alternatives considered)

- **BlendedCost vs UnblendedCost:** Used **BlendedCost** as the default headline metric; use **UnblendedCost** when you care about line-item pricing (e.g. with RIs/Savings Plans).
- **CLI vs console:** CLI is scriptable and repeatable; the console is better for ad-hoc charts and saved reports.
- **No Terraform in this note:** Cost Explorer enablement is account/console workflow; budgets/alerts can be added later as IaC if desired.

## Assumptions

- AWS CLI v2 is installed and credentials resolve (e.g. `aws sts get-caller-identity`).
- Account is a normal payer account with Cost Explorer enabled and data ingested.

## Failure Modes

| Symptom | Likely cause |
|--------|----------------|
| `DataUnavailableException` | CE not enabled yet, or billing data not ingested; wait and retry; try a past closed month once available. |
| `AccessDeniedException` | Missing `ce:*` / billing read permissions on the IAM principal. |
| `ValidationException` … **Group Definition type is invalid** | Use `Type=DIMENSION` in `--group-by`, not `Type=Dimension`. |
| Confusing dates | Remember **`End` is exclusive**; bump `End` one day past the last day you want included. |

## How to Test (commands + steps)

1. Confirm identity: `aws sts get-caller-identity`
2. Run the **monthly total** command (no `--group-by`); expect JSON with `ResultsByTime` and `Total.BlendedCost`.
3. Run the **group by SERVICE** command; expect `Groups` with service names and amounts.
4. Optional: `--output table` on both commands to sanity-check in the terminal.

## Verification checklist

1. Billing console shows **Cost Explorer** enabled and charts load.
2. `aws ce get-cost-and-usage` without `--group-by` returns **200** and a **BlendedCost** amount (or `DataUnavailableException` only until data exists).
3. Same command **with** `--group-by Type=DIMENSION,Key=SERVICE` returns **200** and a `Groups` array.
4. `End` date is **one day after** the last day you intend to include.
5. IAM user/role used by the CLI has billing/Cost Explorer read access.
