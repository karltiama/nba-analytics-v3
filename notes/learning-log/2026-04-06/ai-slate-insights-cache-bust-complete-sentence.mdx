## Context
The slate summary occasionally appeared cut off in UI output (example ending like `(S`), suggesting stale/truncated model text might be served from cache.

## Goal
Force fresh summary generation after prompt updates and reduce risk of incomplete endings.

## Files Changed
- `app/api/betting/ai-slate-insights/route.ts`

## Implementation Notes (key logic explained)
- Increased `max_tokens` from `650` to `900` to reduce length-based truncation risk.
- Strengthened prompt instruction to require complete sentence endings (no mid-word/mid-parenthesis endings).
- Added cache key version segment (`promptVersion = 'v2-complete-sentence'`) to bust prior cached responses and ensure updated prompt behavior takes effect immediately.

## Key Decisions (and alternatives considered)
- Chose cache-key versioning because stale cached output is a likely source when prompt changes don’t reflect right away.
- Alternative considered: disabling cache entirely; rejected to avoid unnecessary API cost and latency.

## Assumptions
- Existing truncated output may have been cached under old prompt key.

## Failure Modes
- Model can still produce weak phrasing if source context is noisy.
- If provider changes behavior, `max_tokens` alone may not guarantee ideal completion quality.

## How to Test (commands + steps)
1. Open `/betting` for the same date where truncation happened.
2. Hard refresh to force a new route request.
3. Confirm summary no longer ends mid-word/mid-parenthesis.
4. Change date and switch back; confirm stable complete endings across requests.

## Next Steps
- If truncation persists, add finish-reason inspection and one retry-on-length in the API route.
