## Context
Email/password sign-in path performed `router.refresh()` before `router.push()`, which can add extra work and perceived delay.

## Goal
Speed up post-auth navigation by removing redundant refresh on successful sign-in.

## Files Changed
- `app/login/LoginClient.tsx`

## Implementation Notes (key logic explained)
- Removed `router.refresh()` after successful `supabase.auth.signInWithPassword`.
- Kept `router.push(nextPath)` as the sole post-login navigation action.

## Key Decisions (and alternatives considered)
- Chose minimal change with clear UX payoff.
- Alternative considered: replace with `router.replace(nextPath)`; deferred because push behavior may be desirable for navigation history.

## Assumptions
- Session cookies/tokens are available without forcing a route refresh before navigation.

## Failure Modes
- In rare edge cases of delayed cookie propagation, the destination route may still briefly require auth check/redirect loop.

## How to Test (commands + steps)
1. Open `/login`, sign in with valid credentials.
2. Measure time from submit click to landing on `/betting`.
3. Verify no functional regression (user appears signed in in header).
4. Verify invalid credentials still show proper error.

## Next Steps
- If delay persists, dedupe duplicate `/api/user/profile` calls between `Header` and `OnboardingGate`.
