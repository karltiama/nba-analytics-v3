## Context
After removing deterministic cards from the AI sidebar panel, `app/betting/page.tsx` still conditionally showed a skeleton based on `loadingInsights`.

## Goal
Render `AIInsightPanel` consistently regardless of `loadingInsights`, since panel value now depends on slate summary state.

## Files Changed
- `app/betting/page.tsx`

## Implementation Notes (key logic explained)
- Removed conditional rendering (`loadingInsights ? skeleton : panel`) inside the sidebar.
- Sidebar now always renders `AIInsightPanel`.
- The panel’s own `slateSummaryLoading` state handles loading UX for summary content.

## Key Decisions (and alternatives considered)
- Chose always-render to avoid unnecessary placeholder flashes tied to unrelated insights fetch timing.
- Alternative considered: keep conditional but switch to `slateSummaryLoading`; rejected to avoid duplicate loading handling.

## Assumptions
- Sidebar should remain visible and stable while summary loads.

## Failure Modes
- If `AIInsightPanel` later depends on data from `/api/betting/insights`, it may need a dedicated loading state.

## How to Test (commands + steps)
1. Open `/betting` with cache disabled or hard refresh.
2. Confirm sidebar panel appears immediately (not hidden by insights skeleton).
3. Confirm summary loading spinner appears until text resolves.
4. Confirm no regression in main `BettingInsights` section behavior.
