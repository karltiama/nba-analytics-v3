## Context
The sidebar summary still required scrolling inside the summary block, even after increasing panel height.

## Goal
Use panel whitespace for summary content first, and remove inner summary scrolling.

## Files Changed
- `components/betting/AIInsightPanel.tsx`

## Implementation Notes (key logic explained)
- Removed the summary-specific wrapper (`max-h-52 overflow-y-auto`) so text is not capped in a nested scroll region.
- Changed summary section container to `flex-1` with `pb-3` so it can occupy available vertical space inside the panel.
- Kept panel-level overflow behavior intact for edge cases with very long content.

## Key Decisions (and alternatives considered)
- Chose removing nested scroll to improve readability and reduce “scroll inside scroll” friction.
- Alternative considered: increasing `max-h-52` to a larger value; rejected because it still preserves nested scrolling.

## Assumptions
- Preferred UX is to read most summaries without interacting with a dedicated mini-scroll area.

## Failure Modes
- Extremely long summaries can still require panel-level scrolling.
- If summary text is short, extra whitespace may appear below text (expected by request).

## How to Test (commands + steps)
1. Open `/betting` and load a date with a generated summary.
2. Confirm summary block no longer has its own scroll behavior.
3. Confirm summary uses more vertical whitespace and shows more text by default.
4. Confirm panel remains stable and readable at different viewport heights.

## Next Steps
- If you want even more whitespace-first behavior, increase sidebar min height to `75vh`.
