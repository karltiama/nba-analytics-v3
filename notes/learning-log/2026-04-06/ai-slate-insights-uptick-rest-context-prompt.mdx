## Context
Slate summary output in the betting sidebar felt generic (pace/ORTG/DRTG heavy) and less useful for prop workflow.

## Goal
Shift AI slate summaries toward actionable context: likely player uptick spots and regular-season rest/rotation risk framing.

## Files Changed
- `app/api/betting/ai-slate-insights/route.ts`

## Implementation Notes (key logic explained)
- Updated the system prompt to prioritize:
  - player uptick opportunities from provided trend data
  - late-season rest/rotation uncertainty context
- Explicitly de-prioritized generic pace/offense/defense recitation unless needed for one clear matchup point.
- Increased `max_tokens` from `400` to `650` to reduce truncation risk and complete thoughts.
- Added a required closing caveat about monitoring confirmed lineups/active status near tip.

## Key Decisions (and alternatives considered)
- Chose prompt-level behavior control first because it is the lowest-risk, smallest diff.
- Alternative considered: changing payload construction in `ai-slate-context`; deferred to keep this change focused and easy to validate.

## Assumptions
- Trending player section is populated often enough to support uptick-focused summaries.
- Users prefer concise context over broad season-stat recap.

## Failure Modes
- If trend data is sparse, summary may still be generic but should remain concise.
- If upstream data lacks lineup context, rest/rotation notes remain high-level.

## How to Test (commands + steps)
1. Run app and open `/betting`.
2. Load a date with active games and verify summary generation.
3. Confirm summary discusses player trend/uptick context first.
4. Confirm summary includes a lineup/active-status caveat.
5. Confirm summary avoids long pace/ORTG/DRTG recitation.

## Next Steps
- If outputs still feel generic, trim pace/defense lines from `lib/betting/ai-slate-context.ts`.
- Add a small prompt regression test with snapshot assertions for style/priority.
