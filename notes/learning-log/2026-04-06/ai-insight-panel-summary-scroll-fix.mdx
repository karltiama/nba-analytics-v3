## Context
The betting sidebar `AIInsightPanel` showed a truncated-looking slate summary paragraph in a constrained sticky panel.

## Goal
Ensure the slate summary text can be fully read without visual clipping while preserving the panel layout.

## Files Changed
- `components/betting/AIInsightPanel.tsx`

## Implementation Notes (key logic explained)
- Removed `shrink-0` from the summary wrapper so it does not force awkward clipping behavior in the flex column.
- Wrapped summary text in a bounded scroll container (`max-h-52 overflow-y-auto`) so long text remains fully accessible.
- Kept text rendering as `whitespace-pre-wrap` to preserve model formatting.

## Key Decisions (and alternatives considered)
- Chose an internal scroll region for only the summary block to keep insight cards visible below.
- Alternative considered: remove all height constraints from the card; rejected because it can overflow viewport and hide footer/actions.

## Assumptions
- Summary length can vary and may exceed available sidebar space.
- Users still need insight cards visible in the same panel.

## Failure Modes
- If summary text is extremely long, users must scroll inside the summary block.
- On very small screens, nested scrolling could feel slightly less smooth.

## How to Test (commands + steps)
1. Run app and open `/betting`.
2. Ensure a date with generated slate summary is selected.
3. Confirm long summary text is fully readable via summary block scroll.
4. Confirm insight cards still render below the summary.
5. Confirm footer remains visible and panel does not break layout.

## Next Steps
- Tune summary max height based on UX feedback (e.g. `max-h-44` or `max-h-60`).
- Consider a “Show more / Show less” toggle for mobile.
