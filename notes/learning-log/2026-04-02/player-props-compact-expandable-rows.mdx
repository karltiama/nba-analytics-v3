## Context

Matchup **Player props** table listed every market per player, producing very tall DOM (~1500px+) and poor scanability.

## Goal

Default to **one row per player** (primary market), with **per-player expand** to reveal additional lines; keep existing filters; cap vertical growth with scroll.

## Files Changed

- `components/betting/MatchupPageLayout.tsx` — `PlayerPropsFilterableList`

## Implementation Notes (key logic explained)

- After applying Player / Prop / Line filters, props are **grouped by `playerId`** and sorted by `propType` then `lineValue`.
- **Primary row:** `points` prop when present, else first row in sorted order.
- **Chevron** only when the player has **more than one** row after filters; badge `+N` for extra markets.
- Expanded rows use a muted **↳** first column and light row background.
- Table wrapper: **`max-h-[min(520px,65vh)] overflow-y-auto`** so the card stays bounded.
- Header summary: `N players · M lines` plus hint text.

## Key Decisions (and alternatives considered)

- **Primary = points:** Matches typical betting interest; footnote explains behavior.
- **Per-player expand vs global “compact mode”:** Expand avoids losing detail for one-off deep dives without a mode toggle.
- **No “expand all”:** Keeps first iteration small; add later if needed.

## Failure Modes

- Duplicate `(propType, lineValue)` rows from data could confuse `samePropMarket` — rare.

## How to Test (commands + steps)

1. `npx tsc --noEmit`
2. Open matchup with many props — one row per player by default; chevron opens rest.
3. Filter to one prop type — chevrons disappear when only one line per player.
