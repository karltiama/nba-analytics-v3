## Context
Props Explorer showed a wide table including a Game id column, a single EV column for the selected track, plus redundant EV A and EV B diagnostic columns.

## Goal
Remove Game, EV A, and EV B from the table; keep the primary EV column and align empty-state `colSpan` with the visible column count.

## Files Changed
- `app/betting/props-explorer/page.tsx`

## Implementation Notes (key logic explained)
- Dropped `<th>` / `<td>` for Game (game id link), EV A, and EV B.
- Set loading / empty `colSpan` to **14** (Player, Prop, Side, Line, Book, Odds, Implied, Conf, Model, EV, Proj, Updated, Save, Paper).
- Removed `ev_track_a` / `ev_track_b` from the sort dropdown; API still accepts `sort=ev_track_a|ev_track_b` if bookmarked.
- Trimmed `ExplorerRow` fields that were only used for the removed cells (`evTrackA`, `evTrackB`, track model fields).
- Subtitle copy no longer mentions “Track A/B reference columns”; sort label for EV is shortened to “EV”.

## Key Decisions (and alternatives considered)
- Decision: leave API response shape unchanged (extra JSON fields are harmless).
  - Alternative: strip `evTrackA` / `evTrackB` from the route for smaller payloads (deferred; not requested).

## Assumptions
- Users can still filter by game via the slate dropdown; losing the per-row game id link is acceptable.

## Failure Modes
- Bookmarks with `sort=ev_track_a` still work but the control no longer lists those options (URL-only).
