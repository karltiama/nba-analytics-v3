## Context

Matchup game pages were a single long column with a sticky header and horizontal section pills, heavy vertical spacing, and duplicate section titles (page `h2` plus card titles).

## Goal

Add a **desktop sidebar** for in-page navigation, **compact** vertical rhythm, and **remove duplicate headings** without changing data fetching.

## Files Changed

- `components/betting/MatchupPageLayout.tsx`

## Implementation Notes (key logic explained)

- **`lg:` layout:** `flex-row` with `aside` (`w-[13.75rem]`, `sticky top-4`) listing the same `SECTION_IDS` as vertical buttons; main column `flex-1 min-w-0 space-y-4`.
- **Mobile / tablet:** Horizontal pill row remains under the header; **`lg:hidden`** on that row so it does not duplicate the sidebar on large screens.
- **Header:** Slightly tighter padding and typography (`py-3`, smaller calendar icon).
- **Scroll offset:** `scrollToSection` uses a smaller offset on `lg` (100 vs 150) because the sticky header is shorter without a second pill row; section `scroll-mt` set to `scroll-mt-28 lg:scroll-mt-24`.
- **Removed** redundant `h2` above Historical matchups, Key player matchups, Injuries, and the extra **Player props** title outside the glass card.

## Key Decisions (and alternatives considered)

- **Sidebar left:** Matches common dashboard patterns; right rail reserved for future (e.g. bet slip).
- **No new dependencies:** Pure Tailwind + existing state.

## Assumptions

- `lg` (1024px) breakpoint is acceptable for sidebar visibility.

## Failure Modes

- Very small `lg` viewports: sidebar + main may feel tight — width is fixed ~14rem; main still fluid.

## How to Test (commands + steps)

1. `npx tsc --noEmit`
2. Resize: &lt;1024px — horizontal pills; ≥1024px — left **On this page** nav, no pills in header.
3. Click each nav target; section should scroll into view without hiding under the header.
