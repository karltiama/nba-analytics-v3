## Context

Game context (injury teammate splits) was table-first; product asked for assistant-style paragraphs instead of raw grids.

## Goal

Render a **deterministic prose summary** (no LLM API) plus collapsible **underlying numbers** tables.

## Files Changed

- `components/betting/MatchupPageLayout.tsx` — `buildInjuryContextNarrative`, `useMemo` for narrative, UI: intro + bordered paragraphs + `<details>` for tables.

## Implementation Notes (key logic explained)

- **Tone:** First-person hedging (“I would flag…”) and explicit limitations; not claiming model inference.
- **Content:** One intro paragraph; one paragraph per injury entry; highlights largest |Δ PTS| teammate, optional second sorted teammate; `low_sample` adds caution.
- **Tables:** Unchanged columns, moved under “Underlying numbers”.

## Key Decisions (and alternatives considered)

- **No LLM:** Avoids latency, keys, and non-determinism; copy can be swapped later for a real generator.
- **`<details>`:** Keeps proof for power users without dominating the card.
