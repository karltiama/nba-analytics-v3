## Context
The /betting dashboard had a today's games grid and AI insights sidebar but no player-level trending overview in the landing viewport. Users care about both team-level and player-level analysis, and we wanted to surface players outperforming their recent baseline at a glance.

## Goal
Add a compact, horizontal-scrolling "Trending Players" strip above the Today's Games section on the betting dashboard. Definition of trending: `trend_score = L5_avg - season_avg` (positive only). Support filtering by stat category (PTS, REB, AST, 3PM, PRA).

## Files Changed
- `lib/betting/queries.ts` — added `TrendingStripPlayer` type, `TrendingStat` type, and `getTrendingPlayersStrip()` query function
- `app/api/betting/players/trending-strip/route.ts` — new GET endpoint with `?stat=` and `?limit=` params
- `components/betting/TrendingPlayerStrip.tsx` — new self-contained client component (cards, tabs, skeleton, empty state, scroll controls)
- `components/betting/index.ts` — barrel export for the new component
- `app/betting/page.tsx` — imported and placed `<TrendingPlayerStrip />` above Today's Games

## Implementation Notes
- The query uses a single CTE-based SQL statement that computes L5 averages from `analytics.player_game_logs` and season averages from `analytics.player_season_averages`. Since `player_season_averages` has no `fg3m_avg` column, 3PM season avg is computed inline from game logs via a separate CTE (`season_3pm`).
- Next opponent is derived by joining `bbref_schedule` (future games only, ordered by nearest date) to the player's current team.
- Only players with positive trend_score for the selected stat are returned (outperforming baseline), sorted by absolute trend magnitude.
- The component is fully self-contained — it manages its own fetch, loading, and error states. Integration into the page is a single JSX tag.
- Horizontal scroll uses CSS snap scrolling with JS-driven left/right arrow buttons and gradient fade edges.

## Key Decisions (and alternatives considered)
- **Separate API endpoint** (`/trending-strip`) vs extending `/trending` — chose separate to keep the compact shape clean and avoid breaking the existing PlayerCard data contract.
- **Self-contained component** vs lifting state to the page — chose self-contained to minimize integration footprint and keep the page file small.
- **Positive-only trends** — only showing players trending UP, not down. The strip is meant to highlight outperformance, not underperformance. Could be toggled later.
- **Raw difference** (`l5 - season`) vs percentage — raw difference is more intuitive for readers ("+4.2 PTS" is clearer than "+18%").

## Assumptions
- `analytics.player_game_logs.pra` and `analytics.player_season_averages.pra_avg` columns are populated by the existing nightly pipeline.
- The `bbref_schedule` table has future games seeded for the current season.
- Players must have `games_played >= 5` and `pts_avg >= 10` to appear (matches existing trending filter).
- Analytics `player_id` values work with the `/betting/players/[playerId]` route's ID resolution.

## Failure Modes
- If the API call fails, the component catches the error and shows an empty state (no crash).
- If `bbref_schedule` has no future games, `next_opponent_abbr` will be null and the card gracefully omits the "vs OPP" text.
- If the `pra` or `three_pointers_made` columns are NULL for some players, COALESCE/fallback logic handles it (the AVG aggregation ignores NULLs).
- If no players meet the positive-trend threshold for a given stat, the empty state message is shown.

## How to Test
1. Start the dev server: `npm run dev`
2. Navigate to `http://localhost:3000/betting`
3. Verify the "Trending Players" strip appears above Today's Games
4. Click each stat tab (PTS, REB, AST, 3PM, PRA) and confirm cards re-render with different players/values
5. Verify horizontal scroll arrows work and gradient fades appear/disappear correctly
6. Click a player card and confirm it navigates to `/betting/players/<id>`
7. Inspect a card: confirm rank, name, team abbr, L5 avg, and "+X.X" diff are displayed
8. Test the API directly: `GET /api/betting/players/trending-strip?stat=ast&limit=5`
