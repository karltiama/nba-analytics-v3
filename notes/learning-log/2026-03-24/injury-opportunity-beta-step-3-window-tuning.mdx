## Context
Initial endpoint sanity check returned zero candidates, likely because the game selection window was too narrow for current schedule timing.

## Goal
Tune only the game window to improve MVP usability while preserving conservative modeling guardrails.

## Files Changed
- `lib/betting/queries.ts`

## Implementation Notes (key logic explained)
- Adjusted upcoming game filter window:
  - from: `now()` to `now()+2 days`
  - to: `now()-6 hours` to `now()+7 days`
- This keeps the feature “upcoming-focused” but avoids empty outputs due to timing gaps.

## Key Decisions (and alternatives considered)
- Chose minimal scope tuning (time window only), not model formula changes.
- Alternative considered: removing time window entirely; rejected to keep beta signal focused on actionable slates.

## Assumptions
- Users want candidate visibility for near-term slate exploration, not only same-day games.

## Failure Modes
- If no scheduled games in 7-day range, endpoint can still return empty list.

## How to Test (commands + steps)
