## Context

The betting dashboard showed a **Players to Watch** grid (L5 vs season trends) below **Betting Insights**, overlapping conceptually with **TrendingPlayerStrip** and adding redundant API work.

## Goal

- **Goal:** Remove the redundant “Players to Watch” section from the betting page.
- **Inputs:** User request; `app/betting/page.tsx` implementation.
- **Output/Done:** Section removed; no unused fetch/state for that block; page still loads games, insights, and trending strip.

## Files Changed

- `app/betting/page.tsx`

## Implementation Notes (key logic explained)

- Deleted the **Players to Watch** `<section>` and all **trending players** client state: `players`, `loadingPlayers`, `fetchPlayers`, and the `useEffect` that called it.
- Removed **`ApiPlayer`**, **`transformPlayer`**, and imports **`PlayerCard`**, **`PlayerCardSkeleton`**, **`PlayerData`** only used for this block.
- Dropped unused **`isLoading`** (it was never referenced in JSX).

## Key Decisions (and alternatives considered)

- **Keep `TrendingPlayerStrip`** — still provides a lighter trending-players surface without duplicating the full card grid.
- **Did not delete** `/api/betting/players/trending` — other routes or future use may still call it; removing the route was out of scope.

## Assumptions

- No other code depended on this page fetching trending players on mount (verified via grep to this file only).

## Failure Modes

- If something imported `PlayerCard` only through this page re-export, builds could break — unlikely; `components/betting` barrel still exports `PlayerCard` for other pages.

## How to Test (commands + steps)

1. `npm run build` or `npx tsc --noEmit` (if configured).
2. Open `/betting`: confirm **Players to Watch** is gone; **Trending Players** strip and **Today's Games** / insights still render.
