## Context

The game box score route (`/games/[gameId]`) is a server component that queries the database and can take noticeable time on navigation. Next.js App Router supports a sibling `loading.tsx` file to show UI while the segment is loading.

## Goal

Provide a visual skeleton that mirrors the real game page layout (matchup card, title block, two box-score tables) during client navigations to `/games/[gameId]`.

## Files Changed

- `app/games/[gameId]/components/GameBoxScorePageSkeleton.tsx` â€” layout-matched placeholder UI using the shared `Skeleton` component.
- `app/games/[gameId]/loading.tsx` â€” route segment default export that renders the skeleton.

## Implementation Notes (key logic explained)

- `loading.tsx` is a special file in the App Router: it wraps the `page.tsx` in a React `Suspense` boundary and is shown until the server component for that segment finishes rendering.
- The skeleton reuses the same outer spacing and card styling classes as `page.tsx` (`min-h-screen`, `max-w-7xl`, bordered cards) so the transition feels stable.
- `Skeleton` primitives use explicit `bg-zinc-200 dark:bg-zinc-800` so placeholders read clearly on `bg-zinc-50` / `dark:bg-black` (the base `Skeleton` `bg-white/5` is easy to miss on light backgrounds).

## Key Decisions (and alternatives considered)

- **Dedicated `GameBoxScorePageSkeleton` component** â€” Keeps `loading.tsx` as a one-liner and allows reuse (e.g. future Suspense boundaries inside the page). Alternative: inline skeleton only in `loading.tsx` (fewer files, less reuse).
- **No new dependencies** â€” Uses existing `@/components/ui/skeleton`.

## Assumptions

- Primary use is **client-side navigation** to a game URL; the skeleton appears while the RSC payload loads. A full page refresh may still show a blank shell until HTML streams, depending on Next.js behavior and caching.

## Failure Modes

- If `loading.tsx` fails to compile, the route still works but without a loading state.
- Skeleton dimensions may drift if `page.tsx` layout changes significantly; update the skeleton to match.

## How to Test (commands + steps)

1. `npm run dev`
2. Open the app and navigate to any valid game link (e.g. from home or team schedule) that goes to `/games/<id>`.
3. Confirm a brief flash of the skeleton (matchup block + two table cards) before the real content appears.
4. `npm run lint`
