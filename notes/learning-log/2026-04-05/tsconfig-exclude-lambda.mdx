## Context

Vercel `npm run build` failed: `lambda/player-props-snapshot/worker.ts` imports `aws-lambda` types, which only exist in the lambda package’s `devDependencies`, not at the app root.

## Goal

- **Goal:** Keep Next.js production build green without pulling every Lambda-only type into the root app.
- **Inputs:** Root `tsconfig.json` included `**/*.ts`, so `lambda/**/*.ts` was typechecked.
- **Output/Done:** `exclude: ["lambda"]` in root `tsconfig.json`.

## Files Changed

- `tsconfig.json` — add `"lambda"` to `exclude`.
- `docs/deployment-checklist.md` — note that Lambdas are built/typechecked separately.

## Implementation Notes (key logic explained)

- Next.js app code does not import from `lambda/` (only `scripts/` does, and `scripts` was already excluded).
- Lambda projects keep their own `npm install`, `@types/aws-lambda`, and `tsc` where applicable.

## Verification Checklist

1. `npm run build` succeeds on Vercel / locally.
2. When editing a Lambda, run its local build or `tsc` under `lambda/<name>/` if you use it.
