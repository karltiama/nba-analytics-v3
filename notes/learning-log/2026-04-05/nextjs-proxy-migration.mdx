## Context

Next.js 16 warns that `middleware.ts` is deprecated in favor of `proxy.ts`.

## Goal

- **Goal:** Remove deprecation warning; keep Supabase session refresh + `/betting` auth gate.
- **Inputs:** Existing `middleware.ts` and `lib/supabase/middleware.ts`.
- **Output/Done:** Root `proxy.ts` with named export `proxy`; `middleware.ts` removed.

## Files Changed

- `proxy.ts` — new; exports `proxy` + same `config.matcher` as before.
- `middleware.ts` — deleted (do not keep both — Next.js errors).
- `lib/supabase/middleware.ts` — unchanged (still exports `updateSession`).
- `docs/deployment-checklist.md` — note updated.

## Implementation Notes (key logic explained)

- Next expects **`export async function proxy`** (or default export) from `proxy.ts`.
- **`export const config`** with `matcher` is still supported for limiting which routes run the proxy.
- Proxy runs on **Node.js** by default in Next 16; avoid `export const runtime = 'edge'` in the proxy file (not allowed).

3. After login, `/betting` loads; session cookies still refresh.
