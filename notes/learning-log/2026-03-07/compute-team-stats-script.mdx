## Context
Second step in building team-level derived analytics. The DDL was created in the previous step; now we need the transformation logic to populate both tables.

## Goal
Create an idempotent script that derives `analytics.team_game_stats` and `analytics.team_season_averages` from existing analytics data.

## Files Changed
- `scripts/compute-team-stats.ts` — **new** refresh script

## Implementation Notes
- **Step 1**: Aggregates `analytics.player_game_logs` by `(team_id, game_id)`, joined to `analytics.games` for opponent/home/away/scores. Upserts into `team_game_stats`.
- **Step 2**: Aggregates `team_game_stats` by `(team_id, season)` for averages, win/loss record. Upserts into `team_season_averages`.
- Uses `coalesce(sum(...), 0)` so null individual stats don't break aggregation.
- `points_allowed` comes from `analytics.games.home_score / away_score` (authoritative), not from opponent player logs.
- `result` handles ties (null) and games with missing scores (null) gracefully.
- Win percentage uses `FILTER (WHERE result IN ('W','L'))` to exclude games with unknown results from the denominator.

## Key Decisions
- **Two-pass approach** (game stats first, then season averages) rather than computing season averages directly from player logs — keeps the intermediate table useful for recent games, trend charts, etc.
- **Row-by-row upsert in a transaction** matching existing `compute-player-season-averages.ts` pattern. Could use a bulk CTE upsert for performance, but consistency with existing codebase wins.
- **Own Pool instance** (not importing `lib/db.ts`) — matches all other scripts in the project.

## Assumptions
- `analytics.games.start_time` is populated and can be cast to `date` for `game_date`.
- `analytics.games.home_score` / `away_score` are populated for completed games.
- The DDL from `analytics_team_stats.sql` has been run first.

## Failure Modes
- If DDL hasn't been run, inserts will fail with "relation does not exist".
- If `player_game_logs` has teams not in `analytics.teams`, FK constraint will reject the insert.
- If scores are null for a game, `points_allowed` and `result` will be null (safe, not a crash).

## How to Test
1. Run DDL: `psql $SUPABASE_DB_URL -f db/schemas/analytics_team_stats.sql`
2. Run script: `npx tsx scripts/compute-team-stats.ts`
3. Verify row counts: `select count(*) from analytics.team_game_stats;`
4. Spot check: `select * from analytics.team_game_stats where team_id = '...' order by game_date desc limit 5;`
5. Verify season averages: `select * from analytics.team_season_averages;`
