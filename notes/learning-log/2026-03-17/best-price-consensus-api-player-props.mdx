## Context
We already ingest player props into `analytics.player_props_current` and the player sidebar fetches them from `GET /api/betting/players/[playerId]/props`.

## Goal
Add an Option A foundation for “best price vs market consensus” by enriching the existing props API with consensus and edge fields (no projection model / EV yet).

## Files Changed
- `app/api/betting/players/[playerId]/props/route.ts`

## Implementation Notes (key logic explained)
- Added optional query param `with_consensus=1`.
- For `market_type='over_under'` and `side in ('over','under')`, compute:
  - **Offer implied probability** from `implied_probability` (or from `odds_american` if missing).
  - **Vig-free fair probabilities per sportsbook+line** when both over and under exist at the same sportsbook/line:
    - \(p_{over}^{fair} = p_{over}^{raw} / (p_{over}^{raw} + p_{under}^{raw})\)
    - \(p_{under}^{fair} = p_{under}^{raw} / (p_{over}^{raw} + p_{under}^{raw})\)
  - **Consensus probability** per (game, player, prop_type, line_value, side) as the median of fair probabilities across sportsbooks.
  - **Edge** per offer as `consensusProbability - impliedProbability`.

## Key Decisions (and alternatives considered)
- **Decision**: compute consensus only when `with_consensus=1` to avoid adding overhead to default requests.
- **Decision**: median across sportsbooks for robustness to outliers.
- **Alternative**: compute consensus from raw implied probabilities (simpler) but that bakes in book vig and is less principled than vig-free normalization.

## Assumptions
- `analytics.player_props_current` contains `odds_american` or `implied_probability` for most rows.
- Enough sportsbooks exist for popular markets so a minimum-count median is meaningful.

## Failure Modes
- If a market has <3 sportsbooks with both sides posted, consensus is null and edge is null.
- If odds are missing/invalid, implied probability is null and edge is null.
- Milestone markets will not produce consensus/edge in v1 (no paired sides).

## How to Test (commands + steps)
- Start dev server (if not already):
  - `npm run dev`
- Hit the API with consensus enabled:
  - Open a player page with props loaded, then in browser devtools run:
    - Fetch `/api/betting/players/<playerId>/props?with_consensus=1`
  - Confirm each row includes `consensusProbability` and `edgeProbability` for over/under markets with enough books.
