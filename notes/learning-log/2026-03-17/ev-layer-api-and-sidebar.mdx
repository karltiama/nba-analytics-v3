# EV layer: API and sidebar

## Context

Plan specified adding an EV layer to player props: (1) probability model and (2) model inputs helper were already implemented. This session completed (3) the props API with `with_ev=1` and (4) the sidebar showing EV as primary and Edge as supporting, with Best and sort by EV when available.

## Goal

- **API**: When `with_ev=1`, fetch player model inputs once, compute per-row `modelProbability` and `ev` for over/under rows with a numeric line; attach to response. Rows without model get `modelProbability: null`, `ev: null`.
- **Sidebar**: Request props with `with_ev=1`; show EV % as primary and Edge as secondary; Best = max EV when any row has ev, else fallback to edge/odds; sort by ev desc then edge desc.

## Files Changed

- `app/api/betting/players/[playerId]/props/route.ts` — Added `with_ev` param, `getPlayerPropModelInputs` + `getStatsForPropType` + `computePlayerPropProbability`, decimal-from-American fallback, and `modelProbability`/`ev` on each prop.
- `components/betting/PlayerPropSelectorSidebar.tsx` — Added `with_ev=1` to fetch; extended `PlayerPropRow` with `modelProbability` and `ev`; added `formatEv`; `groupsWithBest` now includes `bestEv` and `hasAnyEv`; sort by ev then edge; display EV primary, Edge secondary; Best by EV when `hasAnyEv`, else by edge/odds; "Best edge only" filter respects Best-by-EV.

## Implementation Notes (key logic explained)

- **API**: `with_ev=1` triggers a single `getPlayerPropModelInputs(playerId)` call. For each row, we only attach model/ev when `market_type === 'over_under'`, side is over/under, and `line_value` is numeric. We use `getStatsForPropType(inputs, prop_type)`; if it returns null (unknown prop type or missing stats), we leave `modelProbability` and `ev` null. For **under**, `p_true = 1 - probability`. EV formula: `ev = p_true * odds_decimal - 1`. Decimal odds are taken from `odds_decimal` when present and finite; otherwise derived from American: `american < 0 ? 1 + 100/|american| : 1 + american/100`.
- **Sidebar**: `bestEv` is the max `ev` in the group (or -Infinity if none); `hasAnyEv` is true if any row has finite `ev`. When `hasAnyEv`, Best is the row(s) with `ev === bestEv`; otherwise Best stays by edge then odds. Sort comparator: first by `ev` (desc), then by `edgeProbability` (desc). Display order: "EV {formatEv(r.ev)}" as primary (text-xs), then "Edge {formatEdge(...)}" as secondary (text-[10px] muted).

## Key Decisions (and alternatives considered)

- **Decimal from American in API**: Implemented inline with a small IIFE rather than a shared util; keeps the route self-contained. Alternative: shared `americanToDecimal` in `lib/betting` if reused elsewhere.
- **Best when no EV**: Keep existing behavior (Best by edge, then by best odds) so legacy and non–over/under rows still get a "Best" highlight.
- **formatEv**: Show "—" when ev is null/NaN; otherwise "+X.X%" or "-X.X%" so negative EV is visible.

## Assumptions

- API and sidebar types stay in sync (both expect optional `modelProbability` and `ev` on each prop).
- `getPlayerPropModelInputs` returns null or incomplete stats when player has no games or no season data; we do not attach model/ev in those cases and the UI shows "—" for EV.

## Failure Modes

- If `getPlayerPropModelInputs` throws (e.g. DB error), the whole GET fails with 500; consider try/catch around the EV block and returning props without ev in a future iteration.
- Missing or invalid `prop_type` → `getStatsForPropType` returns null → row gets null ev; UI shows "—".

Check that EV % appears as primary and Edge as smaller/muted; rows with highest EV show "Best"; list is sorted by EV then edge.
