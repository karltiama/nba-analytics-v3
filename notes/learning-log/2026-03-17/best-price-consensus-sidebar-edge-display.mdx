## Context
`PlayerPropSelectorSidebar` already groups props by `(propType, side, lineValue)` and highlights the “Best” odds (max American odds) within each group.

## Goal
Request consensus/edge data from the API and display “Edge vs consensus” per sportsbook offer, highlighting the best offer per line based on edge (when available).

## Files Changed
- `components/betting/PlayerPropSelectorSidebar.tsx`

## Implementation Notes (key logic explained)
- Added `consensusProbability` and `edgeProbability` to the `PlayerPropRow` shape (optional fields).
- When fetching props, the sidebar now adds `with_consensus=1` to the request URL.
- Group selection:
  - If any row in a group has `edgeProbability`, the best row is the one with the maximum edge.
  - Otherwise it falls back to the previous behavior (best American odds).
- Display:
  - Each sportsbook row shows `Edge +X.X%` (or `—` when not computed).

## Key Decisions (and alternatives considered)
- **Decision**: keep the grouping logic unchanged so existing filtering UX stays stable.
- **Decision**: fall back to best odds when edge is unavailable, to avoid empty highlighting for sparse markets.
- **Alternative**: sort rows by edge; deferred to keep diff minimal and avoid surprising reordering.

## Assumptions
- The API returns `edgeProbability` only for eligible over/under markets with enough books.
- `edgeProbability` is small (single-digit %) and best displayed as a percentage with one decimal.

## Failure Modes
- If the API can’t compute consensus (e.g. <3 books with both sides), edges show as `—` and highlight falls back to best odds.
- If `edgeProbability` is present but very close between books, multiple rows may effectively tie; current logic highlights the first max-equal row.

## How to Test (commands + steps)
- With the dev server running:
  - Navigate to a player page with props.
  - Confirm rows show `Edge ...` values for common O/U markets.
  - Confirm the “Best” highlight follows the highest edge when edges exist; otherwise it uses best odds.

## Next Steps
- Add explicit v1 guardrails in UI (e.g. hide edge for `milestone`, only show edge for `over_under` + over/under).
- Consider sorting sportsbook offers within a group by edge desc when edge exists.
