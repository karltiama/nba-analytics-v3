## Context

The `PlayerPropSelectorSidebar` component has three filter controls: stat type dropdown, outcome dropdown (over/under/at least), and a line value +/- input. Users reported that filtering wasn't working.

## Goal

Fix the broken filter logic so all three filters (stat, outcome, line) compose correctly.

## Files Changed

- `components/betting/PlayerPropSelectorSidebar.tsx` — rewrote the `filtered` derivation

## Implementation Notes

The filter callback had outcome checks (`filterOutcome === 'over'`, etc.) that used **early `return`** statements. Because every possible `filterOutcome` value (`'all'`, `'over'`, `'under'`, `'at least'`) hit a return, the line-value filter on the next line was **unreachable dead code**.

Fixed by converting the outcome checks from early-return-true to early-return-false (guard clauses). Now the function flows through all three stages:

1. **Stat filter** — exact match on `propType` (or `'all'`)
2. **Outcome filter** — reject if side doesn't match the selected outcome
3. **Line filter** — reject if line value doesn't match the input

Each stage can only reject (return false), and if none reject, the prop passes.

## Key Decisions

| Decision | Chosen | Alternative |
|---|---|---|
| Guard-clause style (return false) | Yes — composable, each filter is independent | Could use a boolean accumulator (`let pass = true; if (...) pass = false;`) but guard clauses are more readable here |

## Assumptions

- `filterOutcome` only ever holds `'all'`, `'over'`, `'under'`, or `'at least'` (matches the `<select>` options).
- `side` values in the database are `'over'`, `'under'`, or `'milestone'`.

## Failure Modes

- If new outcome options are added to the dropdown without a matching guard clause, those props will pass all filters unfiltered (safe default — shows too much rather than hiding data).
- If `propType` values change casing in the database, the `.toLowerCase()` comparison handles it.

## How to Test

1. Navigate to a player's betting page (e.g., `/betting/players/1629029` for Luka).
2. **Stat filter**: Select "assists" from the stat dropdown → only assist props should appear.
3. **Outcome filter**: Select "Over" → only `side=over` rows. Select "Under" → only `side=under`.
4. **Line filter**: Type `24.5` in the line input (or use +/−) → only props with that exact line value should show. Clear the input → line filter should be removed.
5. **Combined**: Select "points" + "Over" + line `24.5` → should narrow to that exact set of rows.

## Next Steps

- Consider adding a "clear filters" button to reset all three at once.
- The line filter uses exact half-point matching (`roundToHalf`). A range-based filter (±1.5) might be more useful for exploring nearby lines.
