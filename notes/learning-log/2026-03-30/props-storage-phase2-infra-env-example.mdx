## Context
Retention/payload controls need to be visible where deployment env vars are defined.

## Goal
Expose player props payload control env vars in Terraform examples so deployments are consistent.

## Files Changed
- `infra/terraform.tfvars.example`

## Implementation Notes (key logic explained)
- Added optional examples:
  - `STORE_PROP_RAW_JSON = "false"`
  - `PROP_RAW_JSON_SAMPLE_RATE = "0"`
- These defaults align with storage-constrained operation.

## Key Decisions (and alternatives considered)
- Chosen: comment-only guidance in tfvars example.
- Alternative: hardcode values in Terraform resources; rejected to keep environment flexibility.

## Assumptions
- Teams copy this file to `terraform.tfvars` and set values explicitly per environment.

## Failure Modes
- If left unset in prod but expected by operators, behavior may differ from runbook assumptions.

## How to Test (commands + steps)
1. Copy example into `terraform.tfvars`.
2. Set values and run `terraform plan`.
3. Confirm Lambda environment contains configured keys.
