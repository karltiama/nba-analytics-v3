## Context
Raw JSON payload storage needs runtime controls so environments can optimize for storage without code forks.

## Goal
Add validated environment controls for raw JSON persistence and sampling.

## Files Changed
- `lambda/player-props-snapshot/src/env.ts`

## Implementation Notes (key logic explained)
- Extended `LambdaEnv` with:
  - `storePropRawJson: boolean`
  - `propRawJsonSampleRate: number`
- Added parse/validation for `PROP_RAW_JSON_SAMPLE_RATE` in `[0,1]`.
- Added boolean parse for `STORE_PROP_RAW_JSON` with default `false`.

## Key Decisions (and alternatives considered)
- Chosen: strict validation with startup failure on invalid values.
- Alternative: silently clamp invalid values; rejected to avoid hidden misconfiguration.

## Assumptions
- Production defaults to raw JSON disabled unless explicitly enabled.

## Failure Modes
- Invalid sample rate string causes lambda startup error.

## How to Test (commands + steps)
1. Set `STORE_PROP_RAW_JSON=false` and run lambda locally.
2. Set `STORE_PROP_RAW_JSON=true`, `PROP_RAW_JSON_SAMPLE_RATE=0.1` and run again.
3. Set `PROP_RAW_JSON_SAMPLE_RATE=2` and confirm startup error is thrown.
