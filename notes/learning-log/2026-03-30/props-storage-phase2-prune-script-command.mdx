## Context
Retention pruning script was added, but a standard command makes recurring operations less error-prone.

## Goal
Expose a consistent npm command for 30-day player props v2 pruning workflows.

## Files Changed
- `package.json`

## Implementation Notes (key logic explained)
- Added `prune:player-props-v2` script:
  - `tsx scripts/prune-player-prop-snapshots-v2.ts --days 30`
- This runs in dry-run mode by default unless `--execute` is passed.

## Key Decisions (and alternatives considered)
- Chosen: npm alias for operational consistency.
- Alternative: manual `npx tsx ...` every run; rejected due to command drift risk.

## Assumptions
- Operators will append `--execute` intentionally when ready.

## Failure Modes
- Running command without `--execute` does not prune rows.
- Missing `SUPABASE_DB_URL` causes early failure.

## How to Test (commands + steps)
1. Run:
   - `npm run prune:player-props-v2`
2. Confirm output shows estimate and dry-run notice.
3. Run execute mode:
   - `npm run prune:player-props-v2 -- --execute`
