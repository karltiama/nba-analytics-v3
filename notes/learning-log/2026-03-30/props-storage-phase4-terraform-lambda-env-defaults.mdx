## Context
Player props ingestion added runtime controls for raw JSON storage/sampling, but Terraform did not guarantee these env vars were always present for both worker and controller Lambdas.

## Goal
Ensure Terraform deploys safe default env values for new ingestion controls while preserving override support via `player_props_lambda_env` and controller env maps.

## Files Changed
- `infra/lambda.tf`

## Implementation Notes (key logic explained)
- Updated worker Lambda `environment.variables` merge order to:
  1. defaults (`STORE_PROP_RAW_JSON=false`, `PROP_RAW_JSON_SAMPLE_RATE=0`)
  2. `var.player_props_lambda_env` overrides
  3. infra-managed queue URL
- Updated controller Lambda `environment.variables` similarly with the same defaults and existing lookup-based values.

## Key Decisions (and alternatives considered)
- Chosen: set defaults in Terraform to avoid missing env drift across environments.
- Alternative: rely on application defaults only; rejected for weaker operational consistency.

## Assumptions
- Teams may override defaults in `terraform.tfvars` when they explicitly want sampled/raw JSON capture.

## Failure Modes
- Incorrect override values (e.g. invalid sample rate) can still fail lambda startup due to env validation.

## How to Test (commands + steps)
1. Run `terraform -chdir=infra plan -var-file=terraform.tfvars`.
2. Confirm only Lambda environment variable diffs for player props worker/controller.
3. Apply and verify env vars in AWS Lambda console for both functions.
