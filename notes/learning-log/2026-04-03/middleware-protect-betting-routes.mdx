## Context

Betting UI was public while `/api/user/*` required auth. Product requirement: all `/betting` app routes require a signed-in user.

## Goal

- **Goal:** Redirect anonymous users from `/betting` and `/betting/*` to `/login` with a safe `next` return URL (including query string).
- **Inputs:** Next.js middleware + Supabase session from cookies.
- **Output/Done:** Unauthenticated access to betting pages becomes a login redirect.

## Files Changed

- `lib/supabase/middleware.ts` — after `getUser()`, if path is betting and `user` is null, `NextResponse.redirect` to `/login?next=...`; copy cookies from the session refresh response onto the redirect so Supabase cookie updates are not dropped.

## Implementation Notes (key logic explained)

- **Path match:** `pathname === '/betting'` OR `pathname.startsWith('/betting/')` covers the segment and all nested routes; does **not** match `/api/betting/*`.
- **`next` param:** `${pathname}${request.nextUrl.search}` preserves `?onboard=1` and other queries.
- **Cookie copy:** Session refresh may mutate `Set-Cookie` on the default `next()` response; those are reapplied on the redirect response when forcing login.

## Key Decisions (and alternatives considered)

- **Middleware vs layout:** Edge middleware avoids flashing protected content and works without a client round-trip.
- **Alternatives:** Client-only `useEffect` redirect — worse for UX and crawlers; skipped.

## Assumptions

- `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` are set so `getUser()` runs; if missing, prior behavior returns `next()` without enforcing (same as before env check).

## Failure Modes

- Misconfigured Supabase env → betting routes may not enforce (early return in middleware).

## How to Test

1. Log out (or incognito). Visit `/betting` → redirected to `/login?next=%2Fbetting`.
2. Visit `/betting/props-explorer` → `next` encodes that path.
3. Sign in → land on `next` path.
4. Logged-in user → `/betting` loads normally.

## Verification Checklist

1. Anonymous `/betting` and child URLs redirect to login.
2. `/`, `/login`, `/signup` still reachable when logged out.
3. `npm run build` passes.
