## Context
Props Explorer users wanted quick access to a player‚Äôs recent form without leaving the slate. The betting player page already loads game logs and charts server-side; there was no small JSON API for client-side preview.

## Goal
Add a sticky right sidebar on `xl+` and a slide-over drawer below `xl` that opens when clicking a player name, showing StatTabs, trend chart (with optional book line), and a compact game log.

## Files Changed
- [`app/api/betting/players/[playerId]/game-log-preview/route.ts`](app/api/betting/players/[playerId]/game-log-preview/route.ts) ‚Äî new GET handler
- [`lib/players/types.ts`](lib/players/types.ts) ‚Äî `propTypeToMetricKey()` inverse of `METRIC_TO_PROP_TYPE`
- [`app/betting/players/[playerId]/components/PlayerTrendChart.tsx`](app/betting/players/[playerId]/components/PlayerTrendChart.tsx) ‚Äî optional `svgWidth` / `chartHeight`
- [`components/betting/PropsExplorerPlayerPanel.tsx`](components/betting/PropsExplorerPlayerPanel.tsx) ‚Äî panel + drawer + placeholder
- [`app/betting/props-explorer/page.tsx`](app/betting/props-explorer/page.tsx) ‚Äî layout, selection state, player cell click + profile icon link

## Implementation Notes (key logic explained)
- **API**: Resolves ID with `resolveAnalyticsPlayerId`, returns `player`, `seasonAverages`, `games` (limit default 25, max 40), plus `resolvedPlayerId` for correct profile links.
- **Metric default**: `propTypeToMetricKey` maps prop types to `MetricKey`; unknown combos fall back to `pts`. Book **line** is passed to chart/table only when `activeMetric` matches that row‚Äôs mapped metric.
- **Viewport**: `useLayoutEffect` + `matchMedia('(min-width: 1280px)')` sets `isXlViewport` so only **one** of sidebar panel or drawer mounts (avoids duplicate fetch).
- **Chart**: Sidebar uses `svgWidth={340}`, `chartHeight={180}`; full player page keeps defaults.

## Key Decisions (and alternatives considered)
- **New API** vs reusing a server action: explicit route keeps the explorer page as a client bundle with a simple `fetch`.
- **External-link icon** next to name: preserves ‚Äúopen full profile in new tab‚Äù without using the name link for navigation.
- **Drawer**: Fixed `right-0` column with backdrop; `xl:hidden` on drawer root.

## Assumptions
- Explorer `playerId` matches values accepted by `resolveAnalyticsPlayerId` (same as rest of betting).

## Failure Modes
- Unresolved player ‚Üí API 404; panel shows error string.
- Very narrow chart labels may crowd; tweak `svgWidth` if needed.

1. `npx tsc --noEmit`
2. Open `/betting/props-explorer`, click a player name ‚Üí sidebar (desktop) or drawer (narrow).
