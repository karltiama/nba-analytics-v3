## Context

Props Explorer had a right-hand sidebar for player preview only. Matchup pages already surface **game context** (Out/Doubtful teammate PTS splits) from `GET /api/betting/games/[gameId]/details`.

## Goal

Show matchup intelligence in the Props Explorer **xl sidebar**, above the player panel, keyed off the **game filter** or the **selected row’s game**.

**Update (same day):** Sidebar now prioritizes **AI Projection Summary** (POST `ai-projection-summary`, same payload shape as the matchup page) with **Underlying numbers** `<details>` preserved for teammate PTS splits. Shared bullet/supplemental builders live in `lib/betting/ai-game-summary-payload.ts`.

## Files Changed

- `lib/betting/injury-context-narrative.ts` — extracted `buildInjuryContextNarrative` (shared with matchup UI).
- `lib/betting/ai-game-summary-payload.ts` — `getGameSummaryBulletsForAi`, `formatOddsHintForAiSummary`, `buildAiSupplementalLines` (matchup + props explorer).
- `components/betting/MatchupPageLayout.tsx` — import narrative + AI payload helpers from lib (remove duplicates).
- `components/betting/PropsExplorerGameContextPanel.tsx` — fetch details + POST AI summary; chips + prose; **Underlying numbers** tables; injury list fallback when no splits.
- `components/betting/PropsExplorerPlayerPanel.tsx` — `PropsExplorerSelection` includes `gameId`; xl sidebar flex height tweaks.
- `app/betting/props-explorer/page.tsx` — `effectiveGameId`; aside layout: game context + player stack.

## Implementation Notes (key logic explained)

- **`effectiveGameId`:** URL `game_id` if set; otherwise `String(selectedPlayer.gameId)` so picking a row loads context for that prop’s game.
- **Panel** reuses the matchup narrative generator and mirrors the “Underlying numbers” `<details>` table pattern in a denser `text-[9px]` layout.
- **Empty split snapshot:** Still show a short explanation plus `injuries.home` / `injuries.away` from the same details payload when useful.
- **Link** “Full matchup” → `/betting/games/[id]`.

## Key Decisions (and alternatives considered)

- **xl-only stacking:** Mobile drawer unchanged (player-only); avoids crowding small screens.
- **Extract narrative to lib:** Single source of truth with `MatchupPageLayout` (no duplicate prose logic).

## Assumptions

- Game ids in props explorer rows match `details` route expectations (same as game page).

## Failure Modes

- Details API 404/500 → panel shows error state; table still usable.
- Large sidebar height: game card `max-h-[min(22rem,42vh)]`, player area `flex-1 min-h-0`.

## How to Test (commands + steps)

1. `npx tsc --noEmit`
2. Desktop (≥1280px): open Props Explorer — sidebar shows game context placeholder until game filter or player selection.
3. Select a game in the filter — context loads for that matchup (splits if data exists).
4. Clear game filter, click a player — context switches to that row’s game.
