## Context

The game matchup page (`MatchupPageLayout`) had a single-column **Odds & line movement** block: current odds, then spread/total line movement charts. The product goal is to pair **sportsbook lines** with **human / prediction-market sentiment** in one section.

## Goal

- Restructure the **Odds & line movement** card into a **two-column layout** on large screens: left = odds + line movement; right = human sentiment.
- Add a typed optional payload (`marketSentiment`) for future Polymarket (or other) integration, with a clear **empty state** until data is wired.

## Files Changed

- `components/betting/MatchupPageLayout.tsx` — types, `MarketSentimentPanel`, grid layout, section copy.
- `components/betting/index.ts` — export `MarketSentimentSnapshot`.

## Implementation Notes (key logic explained)

- **`MarketSentimentSnapshot`**: `homeWinPct` / `awayWinPct` (0–100 scale), optional `source` string for UI attribution.
- **`MarketSentimentPanel`**: If only one side is provided, the other is derived as `100 − x` so a single implied probability can still render. The stacked bar normalizes widths by `awayPct + homePct` so the bar still reads sensibly if the two numbers do not sum to exactly 100 (e.g. rounding).
- **Layout**: `grid grid-cols-1 lg:grid-cols-2` — mobile stacks odds first, then sentiment. A subtle `lg:border-l` separates columns on desktop.
- **Sticky nav label**: `SECTION_LABELS['section-odds']` updated to **Odds & sentiment** so the jump nav matches the new intent.

## Key Decisions (and alternatives considered)

- **Placeholder vs hidden column**: Chose a **visible empty state** so layout and copy are reviewable before any API exists. Alternative: hide the right column until `marketSentiment` is set — fewer UI clues for stakeholders.
- **No new dependencies**: Pure React + existing Tailwind patterns.
- **No API route in this change**: Keeps the diff focused on layout and contract; `/details` can later attach `marketSentiment` from Polymarket or a cache table.

## Assumptions

- Win probabilities are expressed as **percentages 0–100**, not decimals.
- Sentiment is **informational** and labeled as not a sportsbook line (short disclaimer in the panel).

## Failure Modes

- If the API sends invalid numbers (NaN, &lt; 0, &gt; 100), the UI may look odd — validate at the API boundary when wired.
- Very long `source` strings could wrap awkwardly; truncate in the API or add `truncate` if needed.

## How to Test (commands + steps)

1. `npx tsc --noEmit` — should pass.
2. Run the app, open any game matchup page, scroll to **Odds & line movement**.
3. Confirm: two columns on **lg+**; single column on small screens (sentiment below odds).
4. Optional: temporarily pass `marketSentiment: { homeWinPct: 58, awayWinPct: 42, source: 'Polymarket (test)' }` via devtools or a one-line mock in `GameDetailsPageClient` to verify the bar and percentages.
