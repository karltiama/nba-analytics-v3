## Context

Onboarding previously appeared only after `/betting` loaded and `GET /api/user/profile` finished, with no tie to the landing CTA. The hero button ‚ÄúStart Winning Now‚Äù linked to `/signup`.

## Goal

- **Goal:** When a user clicks ‚ÄúStart Winning Now‚Äù, they should see the onboarding modal immediately (or as soon as the betting shell mounts).
- **Inputs:** Landing page CTA ‚Üí navigate to `/betting?onboard=1`.
- **Output/Done:** Modal opens on first paint with a loading shell, then form if the user is signed in and incomplete onboarding; query param is stripped after profile load.

## Files Changed

- `app/page.tsx` ‚Äî ‚ÄúStart Winning Now‚Äù `href` ‚Üí `/betting?onboard=1`.
- `components/betting/OnboardingGate.tsx` ‚Äî detect `onboard=1` once per mount (ref), open modal during profile fetch, strip param after response.
- `components/betting/OnboardingModal.tsx` ‚Äî `phase: 'loading' | 'ready'` for spinner vs form; header copy when loading.
- `app/betting/layout.tsx` ‚Äî `Suspense` around `OnboardingGate` for `useSearchParams` (Next.js requirement).

## Implementation Notes (key logic explained)

- **`onboard=1` is consumed once:** A ref stores whether the first URL had the flag so `searchParams` updates after `router.replace` do not change behavior or retrigger logic incorrectly.
- **Immediate modal:** If launched from CTA, `modalOpen` is true while `loading` is true (`phase === 'loading'`), so the dialog appears with a spinner instead of waiting for the fetch to finish.
- **Strip query:** After profile resolves (success, 401, or error), `onboard` is removed from the URL so refresh/bookmarks stay clean.
- **401:** Modal closes; unauthenticated users still land on `/betting` without a blocking modal (signup flow unchanged for other links).

## Key Decisions (and alternatives considered)

- **CTA ‚Üí `/betting` not `/signup`:** Matches ‚Äústart using the product now‚Äù; logged-out users see the dashboard without onboarding until they authenticate (separate improvement: redirect to login with `returnUrl`).
- **Alternatives:** Open modal on `/` (needs auth detection on marketing page); or post-signup redirect only ‚Äî narrower funnel.

## Assumptions

- Users who should onboard are signed in before or shortly after hitting the CTA, or you will add auth redirect later.

## Failure Modes

- **No Supabase session:** Brief loading shell may flash if `onboard=1` is present, then modal closes after 401.

## How to Test (commands + steps)

1. `npm run build`

3. Complete onboarding ‚Üí click CTA again (if you manually add `?onboard=1`) ‚Üí no form if already completed.
4. Logged out + `?onboard=1` ‚Üí no blocking modal after load.
