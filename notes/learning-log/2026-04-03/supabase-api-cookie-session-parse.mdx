## Context

Signed-in users (e.g. `tiamanat0r@yahoo.com`) could pass middleware but `/api/user/*` returned 401 because `getAuthUserFromRequest` read the wrong cookie shape.

## Goal

Parse `@supabase/ssr` session cookies (`sb-<project-ref>-auth-token`, optional `.0`, `.1` chunks, `base64-` + base64url JSON) so API routes resolve the same user as the browser client.

## Files Changed

- `lib/auth/supabase-user.ts`

## Implementation Notes (key logic explained)

- Discover cookie base names: `sb-*-auth-token` (strip trailing `.\d+` for chunk keys).
- Combine chunked values (base key or `baseKey.0`, `baseKey.1`, …).
- `decodeURIComponent` the combined string (SSR stores URI-encoded chunks).
- If value starts with `base64-`, base64url-decode to UTF-8 JSON; otherwise treat as raw session JSON (legacy array `["jwt",…]` still supported).
- Read `access_token` from parsed object, or first element if array.

## Key Decisions (and alternatives considered)

- **Local decode** instead of importing `@supabase/ssr` internals: avoids deep imports; uses `Buffer` in Node (Edge has `atob` fallback path).
- **Alternative:** Refactor APIs to `createServerClient` + `cookies()` and merge `Set-Cookie` on responses — heavier change; this fix is minimal.

## Assumptions

- Cookie names follow Supabase defaults (`sb-<ref>-auth-token`).
- Session JSON includes `access_token` (current Supabase client behavior).

## Failure Modes

- Custom cookie names or non-default storage → token not found (401).
- Expired access token → `/auth/v1/user` non-OK → null (expected).

## How to Test (commands + steps)

1. `npx tsc --noEmit`
2. Sign in, open `/betting`, confirm Network tab: `GET /api/user/profile` returns 200 with your user (not 401).
3. Optional: DevTools → Application → Cookies → confirm `sb-…-auth-token` (or chunked) exists after login.

## Next Steps

- If 401 persists, log `response.status` from `/auth/v1/user` in dev only to distinguish “no cookie” vs “expired token”.
- Consider migrating user APIs to `createServerClient` for automatic refresh on write routes.
